사업 개요

지역 골목 공실은 단순히 한 점포의 문제가 아니다. 한국부동산원에 따르면 2025년 3분기 전국 중대형 상가 공실률은 13.6%, 비수도권 1층 상가 공실률은 전국 평균을 크게 웃돈다. 그동안의 다양한 활성화 시도들이 단기 효과에 그쳤던 이유는 사람들이 다시 찾아오는 지속가능한 운영 구조를 만들지 못했기 때문이고, 그 본질은 공간마다 그 동네만의 고유성과 이야기가 충분히 담기지 않았기 때문이다.

본 사업은 각 동네의 이야기를 콘텐츠로 담아 사람들이 일부러 찾아오고 싶은 공간을 조성하고, 이를 진입 장벽이 낮은 단기 팝업 형태로 풀어내 임대인·청년 창작자·시민이 모두 부담 없이 참여할 수 있게 한다. 이를 실현하기 위해 본 사업은 6개의 서비스 카테고리로 구성된다 — 공간(공실 단기임대 매칭), 아이템(청년·특산물 큐레이션), 이벤트(페스티벌·야시장 운영), 네트워크·시너지(거리·타운 확산), 멤버십·지역화폐(시민 충성도), 지역 데이터 자산화(정책 근거 축적). 각 카테고리는 독립적으로 작동하면서도 서로의 데이터·사용자·콘텐츠를 공유해 상호 시너지를 만들어낸다.

공간

공실의 단기임대 활성화 / 청년의 무대, 지속가능한 팝업타운

지역에서 비어있는 공간을 청년·창작자의 단기 팝업 무대로 전환하고, 동네의 이야기를 콘텐츠로 함께 풀어내어 시민과 관광객이 반복해서 찾아오는 지속가능한 팝업타운을 조성하는 영역. 임대인과 청년·창작자를 연결하는 매칭·계약·신뢰 시스템과, 동네 자체를 콘텐츠로 보여주는 큐레이션 인프라가 핵심.

임대인 등록·매물 관리 시스템
사진 갤러리 업로드 → 우편번호 API로 주소·좌표 자동 변환 → 가격·일정·연락 방식 입력까지 모바일 5단계 안에 등록 완료. 비즈 메시지·알림톡으로 문의 즉시 수신. IT 도구에 익숙하지 않은 임대인의 진입 장벽을 사실상 0으로.
매물 탐색·매칭 시스템
네이버 지도 SDK 기반 동네 단위 탐색, 평수·가격·기간·업종 필터, 즐겨찾기·찜·알림 구독. AI 추천 엔진(임베딩 기반)이 임차인의 브랜드 성격과 매물의 동네 톤을 매칭해 적합도 점수 산출.
단기 임대 계약·일정 관리 모듈
단기·팝업 특화 표준 임대 양식, 캘린더 기반 일정 충돌 방지, 전자계약 API 연동. 양측 행정 부담을 한 번에 처리.
후기·신뢰 평점 시스템
운영 종료 후 양측 후기 작성, 누적 신뢰 점수, 운영팀 검증 마크. 첫 거래의 심리적 장벽 완화.

아이템

청년 아이템·특산물의 시장 검증

지역 청년 크리에이터·특산물 생산자·소상공인은 좋은 상품이 있어도 도시 소비자를 만날 채널이 없다. 백화점·온라인몰 입점은 진입 장벽이 너무 높고, 인스타·자사몰은 운영 부담이 크다. 단기 팝업 형태로 도시 트래픽 한가운데에서 자기 아이템을 검증할 수 있는 IT 도구가 필요하다.

청년 아이템·지자체 특산물 큐레이션 시스템
생산자가 상품·스토리·제작 배경을 등록하는 통합 DB. 카테고리·태그·시즌별 메타데이터, 검색·필터, 운영팀 큐레이션 워크플로우.
AI 팝업 인테리어·배치 자동 생성 도구
매장 평수·업종·상품군을 입력하면 ML 모델(이미지 생성 + 추천 알고리즘)이 적합한 인테리어 무드보드와 진열 동선을 자동 생성. 작은 브랜드도 디자이너 없이 전문가급 매장을 운영.
디지털 쇼룸 시스템 (3D/360 뷰어)
WebGL 기반 360도 매장·상품 체험. 오프라인 방문 전후 모두 활용. 지역 외부 소비자의 사전 접근.
커머스 모듈
결제 PG·재고·자사몰 연동 API. 팝업 현장의 1회 방문을 자사몰 재구매로 전환. Shopify·Cafe24·자사 백오피스 연동.

네트워크·시너지

Street → Town → 도시 전체의 활성화

지역에서 시작된 좋은 활동이 한 회차로 그치지 않고 다음 단계로 누적되는 구조는 모든 지역 정책의 공통 과제다. 본 사업은 거리 하나의 작은 성공이 옆 거리로 자연스럽게 확산되고, 거리들의 성공이 모여 타운 전체를 살리며, 살아난 타운이 인접 지역과 도시 전체의 활력으로 이어지는 단계적 확산 모델로 설계되어, 한 번의 사업이 끝난 뒤에도 디지털 자산과 운영 노하우가 누적되어 다음 거리·다음 타운·다음 도시의 출발점이 된다.

미션·퀘스트 시스템
입주 브랜드·상점이 함께 만드는 동네 미션. "이 동네 5개 상점 도장 깨기" 같은 시민 참여형 활동으로 유동·체류·재방문 증가. 한 거리의 시민 참여 성공 패턴이 다음 거리로 그대로 이식 가능한 운영 자산.
Street → Town 확산 진척 시각화 도구
거리·타운 단위 성과 누적 시각화. 어디까지 살아났는지, 다음 어디로 확산할지가 한눈에. 지자체의 보고·홍보·다음 거리 선정의 핵심 자료.
통합 운영 대시보드 (지자체·운영팀)
6개 카테고리 데이터를 단일 화면에 통합. React + D3 기반 인터랙티브 차트. 지자체 담당자의 정책 의사결정 보조.
타운 간 데이터 연동 플랫폼 (API)
RESTful API + Webhook 기반으로 거리·타운·인접 지역 간 데이터 연동. 한 거리에서 검증된 운영 표준이 다른 거리·타운·인접 지자체로 그대로 확장되는 기술 기반.
자동 알림·메시지 시스템
성과·이벤트 발생 시 자동 푸시·알림톡 발송. 비즈 메시지·SMS·이메일 채널 통합. 운영팀 수기 부담 제거.

멤버십·지역화폐

1회 방문자를 도시의 팬으로

이 영역이 잘 작동하기 시작하면, 본 사업은 단순한 상권 활성화 사업을 넘어 도시 그 자체의 브랜드와 충성도를 만드는 인프라가 된다. 한 번 다녀간 시민·관광객이 멤버십을 통해 다시 찾고, 동네 곳곳을 누비며 지역화폐로 소비하고, 누적된 활동이 시민의 정체성·소속감으로 이어진다면 — 시민은 도시의 팬이 되고, 관광객은 정기 방문객이 되며, 지자체의 지역화폐는 본 사업의 모든 활동을 잇는 결제·인증 통화로 자리잡는다.

멤버십·등급 관리 시스템
시민·임차인·임대인의 누적 활동 기반 등급. 등급별 혜택·미션 제공. OAuth2 기반 통합 인증·SSO. 활동이 쌓일수록 도시와 시민 사이의 연결이 깊어지는 구조.
쿠폰·포인트 발행·정산 모듈
지역 상점에서 사용 가능한 쿠폰·포인트 발행·QR 정산. 지자체 공동 발행으로 정책 캠페인과 연계 가능.
지역화폐 연동·정산 시스템
기존 지자체 지역화폐와 API 연동. 본 사업 결제 흐름이 곧 지역화폐 사용 확대로 직결. 정책 효과가 사업 활동량에 비례해 확장.
리워드 미션·랭킹 시스템
시민 참여를 게임화. 누적 랭킹·시즌 미션·리워드 자동 지급. 단발성 행사가 아닌 시민의 일상에 동네가 들어가는 지속 참여 흐름.

지역 데이터 자산화

지자체 정책의 영구적 근거 자산

본 사업이 1회성 IT 구축으로 끝나면, 다음 해 정책 결정은 다시 직감과 종이 보고서로 돌아간다. 운영 결과 데이터를 표준화된 형태로 축적해 지자체의 영구적 자산으로 만든다. 향후 정책 결정·학술 연구·민간 협력에 재활용 가능한 지역 데이터셋이 본 사업의 가장 중요한 산출물 중 하나가 된다.

데이터 수집·ETL 파이프라인
공실·매칭·이벤트·소비·유동 데이터의 표준화된 수집·변환·적재. 일배치 + 실시간 스트리밍 혼합 구조.
분석 리포트 자동 생성 엔진
월·분기 자동 발행 PDF 리포트. 템플릿 기반 차트 + LLM 텍스트 생성. 지자체·운영팀 자동 배포.
지자체용 BI 대시보드
별도 학습 없이 직접 사용하는 분석 도구. 드래그·드롭 필터, 시계열·코호트·퍼널 분석.
데이터셋 공개 API·관리 플랫폼
RESTful + GraphQL API, 개인정보 마스킹 자동 처리. 학계·민간 활용 가능한 데이터 게시.
개인정보 비식별화·보안 처리 모듈
개인정보보호법·ISMS-P 기준의 자동 비식별화. 데이터 활용의 법적·윤리적 안전성 확보.

단계적 확장의 그림

본 사업의 IT 인프라는 거리 단위의 작은 성공으로 시작해 옆 거리, 타운, 인접 지역으로 자연스럽게 확산되도록 설계되어 있다. 한 번의 사업이 끝나도 디지털 자산과 운영 노하우, 지역 데이터셋이 누적되어 다음 단계의 출발점이 된다. 거리에서 검증된 운영 모델은 같은 인프라 위에서 타운 단위로 확장되고, 타운의 성공은 인접 지역과 다른 지자체로 옮겨가는 표준 모델이 된다. 이 점이 본 사업이 단발성 IT 구축 사업과 구별되는 가장 큰 차이이며, 지자체가 시간이 갈수록 더 큰 가치를 회수하는 누적형 인프라가 되는 이유다.
