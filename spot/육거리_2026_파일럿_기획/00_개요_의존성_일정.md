# 00. 육거리 2026 파일럿 — 개요·의존성·일정

> 작성일: 2026-05-26
> 대상: 내부 팀 (지자체 보고 자료는 별도)

---

## 1. 한 줄 정의

**2026년 5월 말 ~ 12월 말, 포항 육거리에서 wisetown IT 인프라의 첫 풀스택 검증을 진행한다. 사용자 페이지 / 공실 페이지 / 데이터 시각화 — 세 페이지를 한 데이터 모델 위에 얹어 만들고, 7-8월 야시장 시즌에 1차 검증, 9-10월 "다시 육거리" XL 행사에서 본격 검증한다.**

                              ▲
        ┌─────────────────────┼──────────────────────┐
        │                     │                      │

## 4. 페이지 간 의존성 — 7개 핵심 연결점

| # | 의존성 | 설명 | 결정 시점 |
|---|---|---|---|
| 1 | **공통 익명 ID** | 사용자 페이지의 익명 ID = 공실 페이지 임차인 대시보드의 "방문자 ID" | 데이터 모델 첫 단계 |
| 2 | **QR 위치 ↔ 팝업/공실 ID** | 사용자가 스캔한 QR을 어느 팝업 방문으로 카운트할지 | 공실 등록 시 자동 발급 |
| 3 | **후기 ↔ 팝업 / 임차인** | 사용자 페이지 후기가 임차인 대시보드의 평점, 임대인 데이터 페이지의 평점이 됨 | 1차 출시 시 |
| 4 | **매출 입력 ↔ 데이터 시각화** | 임차인이 입력한 매출이 지자체·투자자 뷰의 K9/K10/K11 KPI가 됨 | 1차 출시 시 |
| 5 | **계약 데이터 ↔ K1/K2/K8** | 공실 등록·계약 체결이 데이터 시각화의 공실률·전환율·신규 입점 수가 됨 | 1차 출시 시 |
| 6 | **카테고리 분류 체계** | 팝업 카테고리(F&B/뷰티/패션/공예/콘텐츠/기타) 표준 정의가 세 페이지에 공유됨 | 데이터 모델 첫 단계 |
| 7 | **권한 모델** | 임대인은 자기 매물·자기 임차인만, 임차인은 자기 팝업만, 운영팀은 전체 | 1차 출시 시 |

### 5.1 마일스톤

| 시기 | 마일스톤 | 산출물 |

| **7월** | 공실 페이지 ★★★ 기능 구현 | 등록·매칭·계약·PG·운영 대시보드 |
| **7월** | 데이터 시각화 v0 | 내부 운영팀 뷰 먼저, 청중 뷰는 v1 |

| **9-10월** | **"다시 육거리" XL 본 검증** | 풀스택 가동. 안전·인파 관리 솔루션 필요성 확인 |
| **11월** | 검증 결과 분석, 다음 시즌 기능 우선순위 | 회고 리포트 |
| **12월** | 결산전·연말 결산, 글로벌 전환 검토 | 이탈리아 도시 1~2곳 파일럿 전환 계획 |

### 5.2 일정상 위험 신호 (Risks)

- **6월 베이스라인 수집 지연** — 9월 "행사 전후 비교"가 불가능해짐. 가장 큰 위험.
- **PG 연동 인증 지연** — 상품 등록·심사에 2~6주. 6월 안에 신청 시작 필수.
- **표준 계약서 법무 검토 지연** — 변호사 검토 일정 미리 잡아야.
- **남녀노소 사용성 검증 부족** — 어르신·외국인 사용자 테스트가 야시장 시즌(7-8월)에 일어나는데 너무 늦음. 6월 중 게릴라 테스트 권장.

## 6. 두 렌즈 분리 — 이번 파일럿에서 어떻게 적용되나

### 렌즈 1 (지자체용·포항)
- **데이터 시각화의 지자체 뷰**가 가장 직접적 산출물. 월간·시즌·연간 PDF 리포트.
- 사용자 페이지·공실 페이지의 "포항 육거리 첫 화면" — 포항 시민 친화 톤·로컬 스토리.
- 카피·이름에 "포항" "육거리" "야시장" 등 노출 OK.

### 렌즈 2 (wisetown 글로벌)
- **데이터 모델·이벤트 스키마**는 처음부터 글로벌. 도시·zone·locale 변수화.
- 사용자 페이지·공실 페이지의 **컴포넌트와 카피 키**는 도시 종속 없는 일반 키로 (`event.title` 같은 i18n 키).
- 데이터 시각화의 **투자자 뷰**는 "이 모델이 다른 도시에서도 작동할까"를 답하는 톤.
- 12월에 이탈리아 도시 1~2곳 파일럿 전환 검토 — 그때 코드 분기 없이 도시 설정만 바꿔서 들어갈 수 있어야.

## 8. 오픈 이슈 (사용자에게 답·결정 필요)

### Q1. 베이스라인 측정 시작 시점
6월 초 QR 설치·익명 수집 시작이 권장이지만, 인프라가 준비되기 전. 임시 측정 방안(예: 단순 카운터, 운영팀 수동 집계)이라도 6월부터 시작할지?

### Q2. 표준 계약서 — 법무 검토 일정
변호사 / 지자체 법무팀 / 외부 법무법인 중 누가 표준 계약서를 검토할지. 일정상 6월 중 시작 필요.

### Q3. PG 선택
포트원(이전 아임포트) / 토스페이먼츠 / KG이니시스 등. 정산 주기·수수료·에스크로 지원 여부 비교 필요. 별도 단기 작업.

### Q4. POS 연동 여부
임차인 매출 입력은 MVP에선 수동(M2)으로 가는 게 빠르지만, 포항 육거리 임차인 중 POS 보유 비율이 높으면 연동 가치 있음. 사전 조사 필요.

### Q5. 회원 가입 정책
비회원도 핵심 기능 사용(D1) + 휴대폰 인증(D2)으로 가입. 카카오 간편 로그인(D3)을 1차에 넣을지 1차 확장으로 미룰지.

### Q6. "다시 육거리"의 안전·인파 관리 (견적서 3-6)
XL 규모면 안전·인파 관리 솔루션이 필요할 수 있는데 이번 파일럿 범위에 포함할지 별도 처리할지.

### Q7. 데이터 시각화 페이지 — 외부 공개 정책
지자체 뷰·투자자 뷰를 외부 URL로 공개할지(임팩트 쇼케이스), 운영팀 권한자에게만 공유할지.

### Q8. 임차인 매출 공유 범위 (I3·M12)
임차인의 매출이 임대인 데이터 페이지에 어디까지 노출되어야 하나 — 디폴트 정책 결정 필요. 임차인 별 옵트인/옵트아웃 + 디폴트는 "범위 노출"로 제안한 상태.

### Q9. 외국인 / 외부 도시 임차인의 본인 인증
사업자등록증·실명 인증을 외국인이 어떻게 통과할지. T3 페르소나 대응 정책.

### Q10. 이번 파일럿의 도메인·브랜드
spot 이름을 유지할지, 공실 페이지·사용자 페이지를 같은 브랜드로 묶을지 분리할지. 두 페이지의 사용자가 완전히 다르므로 브랜드 분리 가능성도 검토.
