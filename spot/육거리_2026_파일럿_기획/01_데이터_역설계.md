# 01. 데이터 — 역설계 기획

> 시작점: **지자체·투자자가 보고 싶어하는 것** → **핵심 KPI** → **수집해야 할 데이터** → **수집 방법**.
> 이 순서로 정해진 데이터 수집 지점이 사용자 페이지·공실 페이지의 UI 동선을 결정한다.

---

## 1. 청중별 보고 싶어하는 것

### 1.1 지자체 (포항시·관련 부처) — 정책 효과 증명

| 관심사 | 구체적 질문 | 보여줄 형태 |
|---|---|---|
| 상권 활성화 | "행사 전 vs 행사 중 vs 행사 후, 거리가 정말 살아났나?" | 시계열 그래프 + before/after 지도 |
| 공실 문제 해결 | "공실률은 줄었나? 단기임대 전환율은?" | 거리 단위 공실률 시계열, GIS 지도 |
| 청년·창작자 유입 | "지역 청년이 들어왔나, 외부 청년이 왔나?" | 임차인 프로필 분포 |
| 시민 만족도 | "포항 시민이 좋아하나?" | NPS, 후기 워드클라우드 |
| 인구·세수·일자리 | "장기적으로 지역 인구·소비·세수에 영향이 있나?" | 카드매출·유동인구 데이터(외부 연동) |
| 정책 ROI | "투입한 예산 대비 효과는?" | KPI 대시보드, 산식 공개 |
| 확산 가능성 | "이 모델을 다른 거리·다른 지역에도 쓸 수 있나?" | 시나리오·A/B 분석 |
| ESG 평가 대응 | "환경·시민 참여·거버넌스 지표가 잡히나?" | ESG 카드 (2026 비중 낮음) |

### 1.2 외부 투자자 — 모델 검증·스케일 가능성

| 관심사 | 구체적 질문 | 보여줄 형태 |
|---|---|---|
| 모델 검증 | "이 모델이 실제로 작동하나?" | before/after 핵심 임팩트 카드 |
| 카테고리 효과 | "팝업 카테고리별 수익 차이는 얼마나 나나? (식음료 vs 화장품 등)" | 카테고리별 매출·방문자 비교 차트 |
| 위치 효과 | "거리 안에서도 위치별 수익률 차이는?" | 거리 내 zone 단위 히트맵 |
| 재방문·리텐션 | "한 번 온 사람이 또 오나?" | 재방문율, 코호트 분석 |
| 스케일 잠재력 | "다른 도시에도 통할까?" | 운영 효율 지표, 표준화 가능성 |
| 비즈니스 임팩트 | "팝업 1개당 얼마 벌고 얼마 운영했나?" | 팝업당 평균 매출·체류 기간 |
| 운영 비용 효율 | "투입 인력·예산 대비 행사 효과?" | 인건비·운영비 대비 KPI |

### 1.3 공통 패턴

세 가지가 거의 모든 청중에게 통한다:
- **Before / After 비교** — 행사 전후 변화량이 가장 임팩트 큼
- **지도 기반 시각화** — "어디서" 변화가 일어났는지 직관적
- **분포·비교 차트** — 카테고리별·위치별·시기별 차이

## 2. 핵심 KPI (15개)

청중별 관심사를 추려서 정한 우선순위 KPI. 데이터 시각화 페이지의 메인 차트는 이 안에서 나온다.

| # | KPI | 정의 | 데이터 원천 | 청중 |
|---|---|---|---|---|
| K1 | 거리 공실률 변화 | (공실 매물 수 / 전체 매물 수), 행사 전/중/후 시점별 | 공실 페이지 등록 데이터 + 외부 부동산 데이터 | 지자체 |
| K2 | 단기임대 전환율 | (단기 계약 체결 수 / 등록된 공실 수) | 공실 페이지 계약 데이터 | 지자체, 투자자 |
| K3 | 누적 방문자 수 | 행사 기간 동안 거리 QR 스캔으로 잡힌 UV | 사용자 페이지 QR | 모두 |
| K4 | 일평균 유동인구 | 시간대별 거리 방문자 수 | QR 스캔 + (가능 시) 외부 통신사 | 지자체 |
| K5 | 재방문율 | 7일·30일 내 재방문한 익명 ID 비율 | 사용자 페이지 로그 | 투자자 |
| K6 | 평균 체류시간 | 한 번 방문 시 머문 시간 | QR 다중 스캔 간격 | 모두 |
| K7 | 시민 만족도 (NPS) | -100 ~ +100 | 사용자 페이지 설문 | 지자체 |
| K8 | 신규 입점 팝업 수 | 행사 기간 입점한 팝업 누적 수 | 공실 페이지 계약 | 지자체 |
| K9 | 팝업당 평균 매출 | 팝업별 매출 합계 / 팝업 수 | 임차인 매출 입력 + PG | 투자자 |
| K10 | 카테고리별 매출 분포 | 카테고리(식음료/뷰티/패션/공예 등)별 누적 매출 비중 | 임차인 매출 + 분류 메타 | 투자자 |
| K11 | 위치별 매출 분포 | 거리 내 zone 단위 매출 비중 | 매출 + 위치 좌표 | 투자자, 지자체 |
| K12 | 임차인 만족도 | 팝업 운영자 NPS | 공실 페이지 설문 | 모두 |
| K13 | 임대인 만족도 | 건물주 NPS | 공실 페이지 설문 | 지자체 |
| K14 | 청년 입점 비율 | 만 39세 이하 임차인 비율 | 임차인 등록 정보 | 지자체 |
| K15 | 콘텐츠 도달 | 로컬 스토리·행사 상세 페이지 뷰·QR 스캔 | 사용자 페이지 로그 | 모두 |

## 3. 수집해야 할 데이터 — 8개 카테고리

### D1. 유동·방문 데이터 (가장 중요)
- QR 스캔 이벤트: `[익명ID, QR지점ID, 타임스탬프]`
- 페이지 뷰·체류 시간 (모바일 웹 행동 로그)
- (선택적) GPS 위치 — 사용자 동의 시에만
- 외부 데이터 (후순위): 통신사 유동인구, 카드매출 매크로

### D2. 사용자 (가벼운 식별)
- 익명 ID (Cookie / Local Storage 기반)
- (선택) 휴대폰 번호 — 회원가입 시
- (선택) 연령대·성별·거주지 — 회원가입 시 추가 입력
- 방문 이력, 후기, 별점

### D3. 행사 마스터·실적
- 행사 ID, 이름, 규모(S/M/L/XL), 시작·종료 시점, 위치 좌표, 카테고리
- 행사별 등록·예약·참여 수
- 행사별 KPI 집계 (K3·K6·K7 등)

### D4. 공실·팝업 마스터
- 공실 마스터: ID, 위치 좌표, 평수, 임대인 ID, 임대료 범위
- 팝업 마스터: ID, 임차인 ID, 카테고리, 시작·종료 시점, 입점 공실 ID
- 공실-팝업 매칭 이력 (한 공실의 N개 팝업 누적)

### D5. 거래·매출
- 임대 계약: 계약 ID, 공실 ID, 임차인 ID, 금액, 기간, 결제 상태
- 팝업 매출: 팝업 ID, 일별 매출, 거래 건수, 객단가 (POS 연동 or 임차인 수동 입력)
- 카테고리 분류 메타데이터 (팝업 카테고리)

### D6. 만족도·피드백
- 시민 별점·후기 (행사·팝업 대상)
- 임차인 NPS (운영 종료 후)
- 임대인 NPS (계약 종료 후)
- 정성 피드백 텍스트

### D7. 콘텐츠
- 로컬 스토리 마스터 (ID, 위치, 카테고리, 글/사진/오디오)
- 콘텐츠별 조회수·QR 스캔수·공유수

### D8. 운영·신뢰
- 인증 데이터 (등기등록부, 사업자등록증)
- 분쟁 신고·처리 이력
- 결제 성공/실패율
- 평균 계약 체결 소요 시간

## 4. 수집 방법 — 어떻게 모을 것인가

### 4.1 Primary collection points (모바일 웹 first)

**① QR 스캔** — 가장 중요한 데이터 수집 채널
- QR 위치 후보:
  - 거리 입구 (zone 단위) → 거리 진입자 카운트
  - 공실/팝업 입구 → 매장 단위 방문
  - 행사장 입구·체크인 데스크
  - 야시장 메뉴/부스 → 메뉴 단위 관심도
  - 로컬 스토리 게시판·노포 앞 → 콘텐츠 조회
- 동작: QR 스캔 → 모바일 웹 즉시 오픈 → 익명 ID 자동 발급(또는 기존 ID 재사용) → 이벤트 기록

**② PG 결제 데이터** — 공실 페이지
- 보증금·임대료 결제 → 계약 금액·결제 시점 자동 수집
- 향후: 행사 티켓·예약 결제도 같은 채널

**③ 임차인 매출 입력** — 공실 페이지 임차인 대시보드
- 옵션 A: POS 연동 (장기적, 우선순위 낮음)
- 옵션 B: **임차인 수동 입력** (단기, 일일/주간 합계만 입력하면 충분) ← MVP는 이쪽
- 카테고리 분류는 팝업 등록 시 결정

**④ 모바일 웹 행동 로그** — 사용자 페이지
- 페이지뷰, 클릭, 체류 시간, 검색 키워드, 스탬프 진행도
- 자체 이벤트 트래커 (GA·Mixpanel·Amplitude 중 결정 — 별도 이슈)

**⑤ 후기·평점·NPS** — 사용자 페이지 + 공실 페이지
- 사용자: 행사·팝업 방문 후 별점 1탭 + 선택적 한줄 후기
- 임차인·임대인: 계약 종료 후 NPS 설문 1회

**⑥ 회원 정보** (선택적)
- 휴대폰 번호 인증 / 카카오 간편로그인
- 추가 정보(연령대·성별·거주지)는 인센티브와 연결 (예: 스탬프 보너스)

### 4.2 Secondary (후순위 / 외부)
- 통신사 유동인구 (KT/SKT/LGU+ 별도 구매)
- 신한·삼성카드 카드매출 매크로 데이터 (별도 제휴)
- 부동산원·실거래가 API

## 6. 데이터 설계 원칙

**① 익명 우선, 식별은 동의 후**
- 모든 사용자는 익명 ID로 시작. 휴대폰 인증·회원가입은 부가 가치 동선에서만 요구.
- 개인정보 비식별화는 별도 모듈(견적서 6-5).

**② 이벤트 스키마는 글로벌 표준으로**
- 모든 이벤트에 `city_id`, `zone_id`, `locale` 필드 디폴트.
- "포항 육거리" 고유 명사는 데이터 모델 레벨에 박지 말 것. `city='pohang', zone='yukgeori'` 같은 코드 값으로.

**③ 카테고리·분류 체계는 처음부터 표준화**
- 팝업 카테고리 (예: F&B / 뷰티 / 패션 / 공예 / 콘텐츠 / 기타) → wisetown 글로벌 공통 분류표 미리 확정.
- 행사 규모 S/M/L/XL → 동시 수용 인원·운영팀 규모 등으로 명시적 정의 필요 (별도 작업).

**④ 청중별 뷰는 같은 데이터 마트에서 분기**
- 지자체용·투자자용·내부용 화면은 권한과 컴포넌트 조합만 다름. 데이터 자체는 한 곳에서.

**⑤ "Before"가 없으면 "After"의 의미가 없다**
- 행사 시작 전 baseline 측정이 필수. 5월말~6월 초에 QR을 거리에 깔아두고 측정 시작해야 9월에 "행사 전후 비교"를 할 수 있음.
- → **6월 베이스라인 수집 시점**이 일정의 첫 마일스톤.

## 7. 데이터 페이지 (운영자 시각화) 구성안

> 청중별 뷰. 동일 데이터 마트, 다른 권한·다른 컴포넌트 조합.

### 7.1 지자체 뷰 (월간/시즌 리포트, 다운로드 가능)
- 헤더: 기간 선택 (월/시즌/연간)
- KPI 카드 6개: K1(공실률), K3(누적 방문), K7(NPS), K8(신규 입점), K14(청년 비율), K13(임대인 NPS)
- Before/After 지도 (거리 zone 단위 색상)
- 시계열: 유동인구 + 매출 (이중 축)
- 카테고리 분포 도넛 차트
- 시민 후기 워드클라우드
- PDF 자동 발행 (견적서 6-2)

### 7.2 투자자 뷰 (임팩트 쇼케이스, 1페이지)
- 가장 위: "행사 전 vs 행사 후" 핵심 임팩트 3개 카드 (큰 숫자 + ▲%)
- 카테고리별 수익률 비교 (식음료 vs 뷰티 vs ...)
- 위치별 매출 히트맵 (거리 평면도 + 색상)
- 재방문율·NPS
- 팝업 단위 효율 (평균 매출 / 평균 체류 기간 / 카테고리)
- 스케일 추정 (이 모델을 N개 거리로 확장하면…)

### 7.3 내부 운영팀 뷰 (실시간 모니터링)
- 실시간 KPI (오늘 방문자, 진행 중 행사, 활성 팝업)
- 이상치 알림 (매출 급감, 분쟁 신고, 결제 실패 급증)
- 모든 raw 데이터 접근 권한
