# 03. 공실 페이지 — 기능 기획

> **대상**: 임대인(건물주)·임차인(팝업 운영자) + 내부 운영팀 (보조).
> **성격**: B2B. 프로페셔널 + 친근. 법적·금전 책임이 동반. PG 연동 포함.
> **목적**: 단기 팝업 임대를 빠르고 안전하게 매칭·계약·정산 → 운영 데이터까지 자동 축적.

---

## 1. 용어·역할 (이 프로젝트 고정 정의)

| 용어 | 정의 | 핵심 동기 |
|---|---|---|
| **임대인** | 건물주 / 공실 소유자 / 단기로 공간을 **내주는** 쪽 | 본인 공실의 운영 데이터(누가 들어왔나, 매출, 만족도)를 보고 다음 임차 결정에 활용 |
| **임차인** | 팝업 운영자 / 청년·창작자·로컬 브랜드 / 단기로 공간을 **빌려 들어가는** 쪽 | 좋은 공실 찾고, 계약·결제를 쉽고 안전하게, 본인 운영 데이터를 한 곳에서 관리 |
| **내부 운영팀** | wisetown 팀 + 지자체 협업자 | 매칭 검증, 분쟁 조율, 데이터 관리, 빠진 빈자리 채우기 |

## 2. 페르소나·시나리오

### 2.1 임대인 페르소나
- **L1. 육거리 1층 상가 건물주, 50~70대** — 디지털 활용 중급. 빈 가게가 오래 가면 손해라는 인식. 단기 팝업은 들어봤지만 누가 들어올지·관리가 귀찮음·세금·계약 안전성 걱정.
- **L2. 다세대 건물 소유 임대업자, 40~50대** — 여러 매물 보유. 효율적으로 관리하고 싶음. 데이터 기반 의사결정 가능.

### 2.2 임차인 페르소나
- **T1. 20~30대 청년 창작자/로컬 브랜드 운영자** — SNS·모바일 네이티브. 짧게 테스트하고 싶음. 계약·세금·법무가 부담.
- **T2. 30~40대 기존 자영업자** — 본점 외에 단기 팝업으로 매출 다각화. 정산·세무 친숙.
- **T3. 외부 도시에서 들어오는 팝업** — 포항 단기 진입. 원격 계약·결제·운영.

### 2.3 시나리오

**SC1. 임대인의 첫 등록**
포항 지자체 안내문 → 공실 페이지 진입 → 사업자 인증 → 건물 등기 업로드 → 매물 등록(평수·사진·임대료 범위) → "신뢰 마크" 획득 → 게시 → 임차 요청 수신 → 임차인 프로필 확인 → 수락 → 표준 계약서 → 전자서명 → PG로 보증금 수신.

**SC2. 임차인의 첫 매물 탐색**
SNS·검색에서 진입 → 사업자/신원 인증 → 카테고리·기간·예산 입력 → AI 추천 매물 → 360° 매물 확인 → 임차 요청 → 임대인과 메시지·일정 협의 → 계약 체결 → 보증금·임대료 결제 → 입주 → POS 또는 수동 매출 입력 → 운영 종료 → 후기·NPS.

**SC3. 임대인의 운영 데이터 확인**
임차 종료 후 임대인 페이지 → "내 공실 데이터" 진입 → 해당 기간 누가 들어왔는지, 방문자 몇 명, 매출 규모 (임차인이 공유한 범위), 시민 후기 별점·텍스트 → "다음 임차인 선정 시 참고" 메모.

## 9. 법무·금융 이슈 메모

- **표준 계약서 (O6)** — 변호사 검토 필요. 한국 부동산임대차보호법·세법 체크.
- **개인정보 (O4)** — 임차인은 사업자 정보·통장 정보 포함 → 별도 처리. ISMS-P 기준 참고.
- **PG (H7/L5)** — 보증금 에스크로 여부 확인. KG이니시스/토스페이먼츠/포트원 등 비교.
- **세금** — 임대 소득은 임대인의 직접 신고 책임. 우리는 매출 데이터·세금계산서 발행 도우미만 (홈택스 연결 정도).
- **분쟁** — 운영팀이 법무 자문 없이 들어가지 않도록 워크플로우 명확화.
- **글로벌 확장** — 한국 법률 모듈을 디폴트로, 이탈리아·기타국 법률은 별도 레이어. 데이터 모델에 `legal_jurisdiction` 필드 미리.

## 10. UX 톤 — "프로페셔널 + 친근"

- 임차인 페이지: 사용자 페이지의 가벼운 톤을 일부 가져옴. 청년이 부담없이 들어올 수 있도록.
- 임대인 페이지: 정보 밀도 높음. 표·숫자 친화적. 어르신 건물주 고려 — 큰 글씨 모드(사용자 페이지 E1)와 동일 토글.
