# Popin — 웹사이트 기획서 (압축본)

> **버전** v0.1 · **작성일** 2026-05-19 · **1차 소스** `PRD.md` v0.2
> 6개 분리 디스커버리(00~06)를 단일 텍스트로 압축. 본 문서가 Claude Design 진입 시 1차 참조.

---

## 1. 한 줄 요약

비어 있는 동네 1층 상가와 단기 팝업을 원하는 브랜드를 잇는 **정보 매칭 플랫폼**. 결제·계약·채팅 모두 외부. 우리는 **검색·등록·문의 수락까지만**. 직방·당근에 가까운 양면 마켓플레이스의 최소 형태.

---

## 2. MVP 가드 — 절대 만들지 않는 것

법적 리스크 회피 + 범위 압축 목적. (PRD §1.2)

- 결제·에스크로·PG
- 전자계약·서명 (표준계약서 PDF 다운로드만)
- 인앱 채팅·메시징 (수락 후 카카오톡·전화로 이관)
- 분쟁 중재 (약관에 "당사자 간 해결" 명시)
- 보증금 보관

이 영역의 화면을 그릴 것 같으면 멈추고 사용자에 확인.

## 3. 사용자 — 양면 균등

**임차인 (트래픽 다수, 디자인 무게중심)** — "김 크리에이터" 29세. 인스타 D2C·로컬 카페·빈티지 셀러. 첫 오프라인·신제품 테스트·팬미팅 목적. 예산 1주 100~500만. **사진이 의사결정의 70%**. 모바일 iOS 비중 높음. 매주~매일 사용.

**임대인** — "박 사장님" 58세. 서울 외곽·경기·지방 1층 상가 보유. 공실 3~6개월. **IT 익숙도 낮음, 카카오톡과 네이버 검색이 전부, 귀찮은 건 못 견딤**. 모바일(안드로이드) + 등록·관리 화면은 데스크탑도 가정. 월 몇 번 사용. **알림톡 필수**.

→ 같은 헤더지만 임차인 모드와 임대인 모드는 메뉴·정보 위계·톤 모두 분기.

## 4. 디자인 가정 (이번 디자인 라운드 한정)

**4.1 반응형 단일 코드베이스.** 데스크탑·모바일 같은 코드 다른 표현. Tailwind 기본 브레이크포인트 (sm 640 / md 768 / lg 1024 / xl 1280). 모든 화면을 모바일 우선으로 먼저 그리고 데스크탑은 폭 활용 분기. 모바일은 "앱처럼" — 하단 탭, 풀스크린 모달, 풀폭 고정 CTA, 스와이프 캐러셀.

**4.2 로그인 후 화면을 1차로.** 비로그인 랜딩·로그인·회원가입은 이번 라운드 제외, 다음 라운드. 임차인 모드 / 임대인 모드 모두 로그인 가정.

**4.3 글자·접근성.** 최소 16px, 본문 17~18px 권장 (임대인 페르소나). 폰트 Pretendard 전 화면 공통.

**4.4 사진 중심·편집 디자인.** 톤은 Appear Here 결. 파랑은 액센트로만, 장식 모션 배제. (memory/design_direction.md)

**4.5 제작 우선순위.**
1. 임차인 핵심 — 탐색 홈 → **타운 페이지** → 검색 결과 → 공간 상세 → 문의 폼 → 수락 후 연락처 공개
2. 임대인 핵심 — 대시보드 → 5단계 등록 → 받은 문의함 → 수락 후 연락처 공개
3. 임차인 마이 (보낸 문의·찜·알림)
4. 비로그인 랜딩·가입 (다음 라운드)
5. 관리자 (별도 라운드, 표준 어드민 라이브러리 활용 권장)

**4.6 외부 API (MVP 확정).**
- **지도** — 네이버 맵 API (Web Dynamic Map). 한국 골목·지하철·POI 정확도 우선. 무료 한도 일 25만 호출 = MVP 충분.
- **지오코딩 (주소→좌표)** — 네이버 Geocoding API. 임대인 등록 시 주소 입력 → 좌표 자동 추출.
- **도로명 주소 검색** — 카카오(다음) 우편번호 API. 사실상 표준·무료. 지도 SDK와 별개로 병행.
- **알림톡** — 카카오 비즈메시지 (NHN 또는 직접 연동, 미정).
- **소셜 로그인** — 카카오 · 네이버 (PRD §3.2 #2).
- React 래퍼는 `react-naver-maps` 등 커뮤니티 라이브러리 사용, TS 타입은 자체 보강.

├── /towns                     팝업타운 목록  ★ 신규

├── /my/inquiries              보낸 문의

├── /host/spaces/:id/edit      공간 수정
├── /host/inquiries            받은 문의
└── /host/profile              임대인 프로필·계정

admin.popin.kr (운영자, 서브도메인 분리)
├── /                          KPI 대시보드
├── /spaces/queue              공간 검증 큐
├── /spaces/all                전체 공간
├── /inquiries                 문의 모니터링
├── /users                     사용자
├── /reports                   신고·CS
├── /messaging                 알림톡 발송 로그
├── /towns                     타운 관리  ★ 신규

**계정·역할.** 계정 하나, 역할 둘 가능. 가입 시 1차 페르소나(임차인/임대인) 선택. 마이에서 반대편 활동 추가 → 양면 활성 시 헤더에 모드 토글. 임차인 = 접두어 없음(트래픽 다수·SEO), 임대인 = `/host`, 운영자 = 서브도메인.

**헤더 메뉴.**
- 임차인 모드: `공간 찾기 · 팝업타운 · 내 활동 · 🔔 · 👤`
- 임대인 모드: `대시보드 · 공간 관리 · 받은 문의 · 🔔 · 👤`
- 양면 활성: 좌측에 `[임차인 ▾]` 모드 토글, 토글에 따라 위 두 헤더 사이를 오감.

## 6. 팝업타운 — 핵심 IA 컨셉

### 6.1 정의

운영팀이 큐레이션하는 **동네 단위 매거진 페이지**. 단순 자치구 필터가 아니라 편집 단위. 예: 성수동 팝업타운, 부암동 팝업타운, 육거리 팝업타운.

**왜.** 임차인이 검색하는 단어는 "성동구"가 아니라 "성수동"·"연남동"이다. 행정 지도와 머릿속 지도가 다름. 또한 팝업은 장소 자체가 콘텐츠 — 동네 톤을 우리가 편집해 보여줘야 인스타·블로그 도는 학습 시간을 우리 안에서 끝낼 수 있다. **"성수동 팝업" 같은 검색어의 SEO 1차 진입점**.

### 6.2 타운 페이지 구성

위에서 아래로 순서대로:

1. 헤로 이미지 (동네 풍경, 16:9 풀폭) + 타운명 + 한 줄 소개
2. 요약 통계 — 공간 N개 / 평균 1주 가격 / 마지막 업데이트
3. 동네 이야기 본문 (200~300자, 편집자 톤. 광고체 X)
4. 동네 분위기 사진 그리드 (6장 정도)
5. 유동인구 패턴 라벨 (평일 낮/저녁/주말 강도, MVP는 운영자 수기)
6. 등록된 공간 카드 리스트 + 필터 칩 (면적·가격·기간)
7. 오는 길 — 미니 지도 + 지하철·정류장
8. 비슷한 타운 추천 (수기)
9. "Popin이 편집했어요" + 마지막 업데이트 일자

**데스크탑** — 헤로 풀폭 후 좌측 매거진(이야기·분위기·차트) / 우측 매물 그리드 2~3열.
**모바일** — 위 순서대로 풀폭 세로 스크롤.

### 6.3 공간 ↔ 타운 매핑

- 1 공간 ↔ N 타운 가능. 임대인이 직접 고르지 않음 (페르소나 부담 ↓).
- 좌표·주소 기반 자동 후보 추출 → 운영자 검증 큐에서 최종 확정 할당.
- 한 공간이 비공개되면 타운 매물 목록에서 자동 빠짐.

### 6.4 다른 화면과의 연결

- **임차인 탐색 홈** — 두 번째 블록 "팝업타운 둘러보기" (단순 칩이 아닌 헤로 카드 3~4개 + "전체 →").
- **검색 결과** — 상단에 "ⓘ 이 결과는 ○○ 팝업타운 페이지에서 동네 소개와 함께 볼 수 있어요 [→]" 배너.
- **공간 상세 위치 섹션** — "이 공간은 [성수동 팝업타운]에 속해요 [타운 보기 →]" 역참조.
- **마이 > 찜한 타운 탭** — 공간 찜과 별도. 신규 공간 등록 알림 기준.

### 6.5 MVP 시작 타운 (잠정, 사용자 확정 필요)

지리형 5~7개 권장. 핫플 위주로만 채우면 차별점(비핫플 로컬) 약화 — 지방 타운 1~2개는 첫 인벤토리에 포함 권장.
잠정 후보: 성수 · 연남 · 망원 · 부암 · 한남 · 육거리 · 광안리.

테마형 타운("코너 통유리", "골목 카페형" 등)은 Phase 2.

### 6.6 라우트·SEO

`/towns/:slug`는 영문 슬러그(SEO 친화) + 페이지 내 한글명 큰 글씨. `/spaces?...` 검색 결과는 `noindex`로 SEO 무게를 타운에 모음. 메타 — title, description, OG 이미지, Schema.org `Place`+`ItemList`.

## 7. 임차인 플로우 (두텁게)

### 7.1 큰 흐름

진입 → 탐색·검색 → 공간 상세 → 문의 작성 → (임대인 응답) → **수락 시 연락처 공개·외부 이관** / 거절·미응답 시 대안 추천. North Star = **문의 발송 건수** (수락·거래는 외부라 측정 불가).

### 7.2 진입 경로

1. SNS 공유 URL → `/spaces/:id` 바로
2. 구글·네이버 검색 ("성수동 팝업") → `/towns/:slug` 타운 페이지 (SEO 핵심 착지)
3. 직접 진입 → 랜딩 → CTA (이번 라운드는 로그인 후라 `/home`으로 가정)
4. 알림 클릭 → 해당 공간·타운으로 딥링크

**원칙:** 어디로 진입해도 즉시 가치 있는 화면. 랜딩에 강제로 거치게 하지 않음.

### 7.3 탐색 홈 (`/home`)

위에서 아래로:
1. 거대한 검색바 ("어디서 팝업 열까요?") + 인기 타운 칩
2. **팝업타운 둘러보기** — 매거진 카드 3~4개 가로 스크롤 + "전체 →"
3. 카테고리별 (Phase 2 — 테마 타운으로 통합 가능)
4. 최근 등록 12개

### 7.4 검색·필터

PRD §4.5 1순위 필터: **지역(자치구) → 면적 → 가격 → 가용시기**. 모바일은 상단 칩 5개(지역·면적·가격·기간·업종) → 탭 시 풀스크린 시트, 적용 후 칩에 값 미리보기. 데스크탑은 좌측 사이드바 인라인.

**리스트가 기본, 지도는 토글** (PRD §4.6). 지도 마커 = 가격 칩 (직방·다방 패턴). 모바일은 우하단 플로팅 "지도로 보기".

**0건/소수 케이스** — 즉시 조건 완화 제안 (가격 +20%, 인근 자치구, 모든 기간). "조건 알림 받기" CTA.

### 7.5 공간 카드 정보 위계

대표 사진 1장(카드 면적 60%) → 지역·층·면적 → 가격 + "협의 가능" 작은 칩 → 태그(선택). 검증 배지는 좌상단 작게: 미검증(회색) / 사진검증(파랑) / 방문검증(진한 파랑·체크).

### 7.6 공간 상세

모바일 정보 위계 (위→아래):
- 사진 캐러셀 풀폭 5장+ (스와이프, 종류 라벨, 핀치 줌)
- 공간명 + 검증 배지
- 가격 + "협의 가능"
- 어코디언: 가용 기간(첫 진입 시 열림) · 공간 정보 · 위치 · 허용/금지 업종 · 임대인 한 줄
- 표준 계약서 다운로드 · 신고하기 (작게)
- **하단 고정 풀폭 CTA "문의하기"**

데스크탑은 우측 sticky 카드에 가격·기간·CTA 묶음.

**가용 기간 캘린더.** 가능(하늘색) · 협의 중(노랑, 7일) · 확정 임차(회색·취소선) · 미지정(흰색·비활성).

**위치.** 네이버 맵으로 **정확 좌표·정확 주소를 그대로 노출** (D13 변경). 지도 위 마커는 공간 위치 핀 + 주변 지하철·정류장 핀. 도보 거리 텍스트 ("성수역 도보 3분"). + **소속 타운 역참조** ("이 공간은 성수동 팝업타운에 속해요"). 모바일은 미니 지도 → 탭 시 풀스크린 지도, 데스크탑은 인라인 지도.

**임대인 정보.** 닉네임/이니셜 + 등록 공간 수 + 평균 응답 시간 + 가입 시기. 전화·카톡은 수락 후에만.

### 7.7 문의 작성 → 발송

폼 필드 (한 화면, 스크롤 짧게):
- **필수** 희망 기간(캘린더), 사용 목적(라디오: 팝업매장·전시·촬영·기타)
- **선택** 자기소개(200자, placeholder에 좋은 예시), 예산(협의 가능)
- 알림톡 동의 체크

폼 하단에 공간 미니 카드 sticky — 어느 공간에 문의 중인지 잊지 않게.

**발송 완료 화면.** ✓ + "보통 6시간 내 응답" + "알림톡으로 알려드릴게요" + [내 문의 보기] [더 둘러보기]. 후자는 검색 결과로 — 한 군데만 문의하고 끝낼 필요 없게.

### 7.8 수락 후 연락처 공개 (가장 큰 단일 화면)

PRD §4.1의 핵심 결정. 이 화면이 평면 텍스트면 안 됨.

```
🎉 박○○ 사장님이 수락했어요
[공간 미니 카드]
임대인 박○○ · 010-XXXX-XXXX
[ 💬 카카오톡으로 대화하기 ]   ← 노란색·큰 풀폭
[ 📞 전화 걸기 ]                ← 큰 풀폭
─ 표준 계약서 ↓
─ 거래 시 주의사항
ⓘ 이후 거래는 두 분이 직접 진행해요. 분쟁은 당사자 간 해결이 원칙입니다.
```

카카오톡·전화 버튼이 화면의 주인공. 카톡 버튼은 채팅방·친구 추가 딥링크. 면책 문구는 작지만 반드시.

### 7.9 거절·미응답

- **거절** — "거절" 단어는 약하게 ("성사되지 않았어요"). 사유 4지선다(강제 X) + 대안 추천 3개 카드.
- **미응답 24h** — 임대인에 재알림톡. 자동 거절 X.
- **미응답 48h** — 임차인에 자동 통보 + 대안 추천. 운영자 모니터링 화면에서 추적.

### 7.10 마이 (`/my/...`)

- **보낸 문의** — 응답 대기 / 수락됨(✨, 카톡·전화 큰 버튼 내장) / 거절됨(대안 추천 동반) / 미응답 자동 분류로 상태별 묶음.
- **찜한 공간·타운** — 탭 2개. 찜한 공간은 가용 변경 알림 종 아이콘, 가격 변경 배지. 찜한 타운은 헤로 카드 + 최근 신규 공간 수 배지.
- **알림함** — 시간 순. 좌측 스와이프 삭제. 상단에 알림 설정 빠른 진입(피로 컨트롤).

### 7.11 모바일 하단 탭

`🏠 홈 · 🔍 탐색 · ♡ 찜 · 👤 마이` — 4개만. 문의함·알림은 마이 하위 탭.

## 8. 임대인 플로우 (압축)

### 8.1 큰 흐름

가입·진입 → **5단계 공간 등록** → 검증 대기 → 노출 → 문의 수신 → 수락 시 연락처 공개·외부 이관 / 거절·미응답. KPI는 응답률 60% + 등록 폼 완료 8분 이내·완주율 70%.

### 8.2 5단계 등록 폼

URL `/host/spaces/new?step=1~5`, **각 단계 임시저장**.

1. 주소 (카카오 우편번호 API로 도로명·지번 검색 → 네이버 Geocoding으로 좌표 자동) — 1분. **"등록한 주소는 공간 상세 페이지에 그대로 공개됩니다" 안내 + 동의 체크박스** 필수.
2. 공간 정보 (면적·층·시설 — 전기·화장실·수도·냉난방·주차·출입) — 1~2분
3. **사진 업로드** ★ 가장 마찰 큼 — 종류별 슬롯 5개 (외관·내부 전경·내부 다른·화장실/주방·자유), 각 슬롯에 안내 + 예시 사진 — 2~3분
4. 가격·기간 (일/주/월 단가, 보증금, 가용 캘린더 드래그 범위) — 1분
5. 업종 조건(허용·금지) + 자기소개 한 줄(200자, 선택) + 알림톡 동의 — 1~2분

UI 원칙: 18px+ · 라벨은 폼 위 큰 글씨 ("주소를 알려주세요") · 한 단계 5필드 이하 · "이전" 버튼 데이터 안 날아간다 표시 · 사진 자동 회전·압축.

수정은 5단계 폼을 어코디언 형태로 재사용 (자유 탐색).

### 8.3 대시보드 (`/host`)

상단 KPI 카드 3 (등록 공간 · 응답 대기 · 이달 조회) → **"빨리 처리할 일"** 블록 (응답 대기 문의를 가장 위, 24h 경과는 ⚠️) → 내 공간 카드 리스트 → 지난 7일 추이.

원칙: 응답 대기를 가장 위. 통계는 보조.

### 8.4 받은 문의함 (`/host/inquiries`)

상태별 묶음: 응답 대기 / 수락 / 거절 / 미응답 자동(48h+).

**응답 대기 카드** — 임차인 이름·인스타 핸들 · 공간 · 희망 기간 · 자기소개 · `[👍 수락]` `[👎 정중히 거절]` 큰 버튼 2개. 카드만 보고 결정 가능해야 함(상세 페이지 들어갈 필요 X).

**알림톡 내 직접 응답** — 카카오 비즈메시지의 버튼으로 "수락/거절" 1탭 가능. IT 익숙도 낮은 페르소나의 핵심 경험.

### 8.5 수락 후 연락처 공개 (임대인 측)

임차인 측 7.8과 거울 화면. 차이: 임차인 인스타 핸들 노출, "거래 시 주의사항"은 임대인 버전 (신원 확인 / 보증금 받기 전 사용 약속 명문화 등 체크리스트).

### 8.6 모바일 하단 탭

`🏠 대시보드 · 🏬 공간 · 💬 문의 · 👤 마이`. 공간 탭에 "+ 공간 추가" FAB.

## 11. 상태 모델 (요약)

### 11.1 공간 (Space)

`작성중` → 5단계 완료 → `검증대기` → 운영자 → `사진검증` 또는 `반려`. 반려는 임대인 수정 후 재제출. `사진검증` 후 운영자 방문 → `방문검증` (Phase 2). `사진검증/방문검증` ↔ `비공개` (임대인 토글 또는 신고). 첫 수락 시 `협의중`(7일) → 만료 또는 거래 보고 시 검증 상태로 복귀. `비공개` → `삭제예정` (30일 후 영구).

노출 정책: `작성중/검증대기/반려/비공개/삭제예정`은 검색 노출 X. `검증대기`는 임대인 본인만 접근. `사진검증` 일반 노출, `방문검증` 상위 노출.

### 11.2 문의 (Inquiry)

`발송됨` → 임대인 열람 시 `임대인확인` → 24h 미응답 시 `미응답24h`(임대인 재알림톡) → 48h 시 `자동통보48h`(임차인 통보 + 대안). 어디서든 `수락` 또는 `거절` 가능. `수락` → 카톡·전화 클릭은 이벤트만(상태 동일). 7일 후 임차인 셀프 보고 트리거 → `거래완료보고` 또는 `종결`. `거절`은 즉시 `종결`.

### 11.3 검증 배지

`미검증` → 운영자 → `사진검증` → 운영자 방문 → `방문검증`. 사진 대규모 수정 시 → `미검증` 자동 강등(임대인 안내). 방문검증은 시간 만료 정책 (예: 6개월, Phase 2 결정).

### 11.4 사용자

`가입대기` → 유형 선택·약관 → `활성` → 90일 무활동 → `휴면` → 재로그인 시 복귀. `활성` → 운영자 정지(기한) → 만료 시 복귀, 또는 영구차단. 본인 탈퇴 시 30일 후 데이터 삭제. 임대인은 응답률 30%×3회 누적 시 `경고` 상태.

### 11.5 타운

`초안`(URL 차단, 운영자 편집) → 슈퍼관리자 게시 → `게시`(공개·SEO) → `비공개`(임시 내림, 매물은 다른 곳에선 계속 노출) → `게시`로 복귀 또는 슈퍼관리자 삭제.

### 11.6 알림톡 발송

`대기` → `발송중` → `발송완료` 또는 `발송실패`(자동 재시도 3회) → 3회 실패 → `발송포기`(운영자 알림). 수신거부 시 SMS 폴백 가능(운영 설정 토글).

### 11.7 신고

`신규` → 운영자 검토 → `처리중` → 결정 (공간비공개 / 임대인정지 / 임차인정지 / 무혐의종결 / 추가확인) → `종결`. 모든 결정은 신고자·피신고자에 알림.

## 12. 알림 트리거 (통합 표)

**임차인 측:**
- 문의 수락 — 알림톡 + 푸시 ("박○○ 사장님이 수락했어요")
- 문의 거절 — 알림톡 (대안 추천 동반)
- 문의 미응답 48h — 알림톡 + 대안 추천
- 찜한 공간 가격 변경 — 푸시 (선택 알림톡)
- 찜한 공간 가용 일정 변경 — 푸시
- 찜한 타운 신규 공간 — 푸시, 주 1회 묶음
- 찜한 타운 페이지 갱신 — 푸시 (편집 시)
- 거래 7일 후 — 알림톡 ("거래는 잘 되셨나요?" 셀프 보고 유도)

**임대인 측:**
- 새 문의 도착 — 알림톡 [수락]·[거절]·[열어보기] 버튼
- 새 문의 후 24h 미응답 — 알림톡 [지금 확인]
- 검증 완료 (사진/방문) — 알림톡
- 검증 반려 — 알림톡 [수정하기]
- 거래 7일 후 — 알림톡 [완료 보고]·[아직]
- 신고로 비공개 — 알림톡 [내용 확인]

**원칙:** 알림톡은 액션 버튼 1~2개로 끝나는 메시지. 줄글 길게 X. 푸시는 보조(PWA). 발송 가능 시간 08:00~22:00 (운영 설정).

## 14. 열린 질문 (사용자 확인 필요)

PRD §9 + 본 디스커버리 신규.

1. **시작 타운 개수·인벤토리** — 5개? 7개? 지방 타운(육거리·광안리 등) 포함 비중?
2. **타운별 헤로 이미지 확보 방식** — 운영팀 직접 촬영 / 임대인 사진에서 차용 / 외주? 첫 인벤토리 7개 × 헤로 1장 + 분위기 6장 = 49장 필요.
3. **유동인구 차트 데이터 소스** — MVP는 운영자 수기 라벨로 잠정. 외부 데이터 연동(SK텔레콤·KT 인구 데이터)은 Phase 2?
4. **임대인 등록 시 사진 누가 찍나** — MVP는 셀프, 운영팀 출장은 Phase 2. UI에 "사진 도움 받기(준비 중)" 자리 잡아둠.
5. **표준 계약서 미리보기 노출 범위** — 잠정: 수락 후만. 미수락 단계엔 다운로드만.
6. **동시 협의 중 표기** — 잠정 7일 노란색 후 자동 복귀. 임대인이 수동으로 "협의 완료" 표시 가능해야 하나?
7. **검증 배지 시각 우선순위** — 잠정: 카드 좌상단 작은 배지 / 상세 사진 오버레이 + 섹션 헤더.
8. **"내 공간 등록하기" CTA로 가입한 사람** — 임대인 기본 선택 + 변경 가능 안내. OK?
9. **테마형 타운** — MVP는 지리형만, 테마형은 Phase 2. OK?

이 중 1~2번이 디자인 작업에 가장 큰 영향 (헤로 이미지 확보가 안 되면 타운 페이지 와이어를 채울 콘텐츠가 없음).
